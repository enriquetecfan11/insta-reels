# Índice de funcionalidades (Features)

Especificaciones de funcionalidades del proyecto en formato Spec-Driven Design. Cada feature debe incluir: **Descripción**, **Requisitos**, **Criterios de Aceptación**, **Implementación**.

## Features documentados

| Feature | Descripción |
|---------|-------------|
| *(ninguno aún)* | Usa `/new-feature <nombre>` para crear una nueva especificación. |

## Cómo añadir una feature

1. Crear `docs/features/NombreDeLaFeature.md` con el [template estándar](index.md#plantilla).
2. Añadir una fila en la tabla de arriba con el nombre y una breve descripción.
3. Mantener este índice actualizado cuando se añadan o eliminen features.

## Plantilla

Cada feature debe tener estas secciones:

- **Descripción** — Qué hace la funcionalidad y por qué existe.
- **Requisitos** — Requisitos funcionales, técnicos, dependencias y restricciones.
- **Criterios de Aceptación** — Qué debe cumplirse para dar la feature por terminada.
- **Implementación** — Enfoque técnico, arquitectura e implementación clave.

Ver el skill SDD o `AGENTS.md` para el flujo completo.
